# Design and Analysis of Algorithms — Bloom Filter та HyperLogLog

Репозиторій містить розв'язки двох завдань з курсу **Алгоритми та структури даних / Робота з даними**:

- `task_1_bloom_filter/` — перевірка унікальності паролів за допомогою фільтра Блума.
- `task_2_hyper_log_log/` — порівняння точного підрахунку унікальних IP-адрес зі структурою HyperLogLog.

---

```text

├── task_2_hyper_log_log/
```

## Запуск завдань

### Task 1 — Bloom Filter для перевірки паролів

```bash
cd task_1_bloom_filter
python3 bloom_filter_passwords.py
```

Скрипт:

- створює екземпляр `BloomFilter`;
- додає список існуючих паролів;
- перевіряє нові паролі на унікальність;
- виводить статус: _«вже використаний»_ / _«унікальний»_ для кожного пароля.

Детальний опис завдання — у файлі  
`task_1_bloom_filter/README_task_1.md`.

---

### Task 2 — HyperLogLog проти точного підрахунку

Перед запуском поклади файл `lms-stage-access.log` у папку `task_2_hyper_log_log/` поруч зі скриптом.

```bash
cd task_2_hyper_log_log
python3 hll_vs_exact.py
```

Скрипт:

1. Завантажує IP-адреси з лог-файлу.
2. Обчислює точну кількість унікальних IP-адрес за допомогою `set`.
3. Обчислює наближену кількість за допомогою `HyperLogLog`.
4. Порівнює час виконання та виводить таблицю з результатами й відносною похибкою.

Детальний опис завдання — у файлі  
`task_2_hyper_log_log/README_task_2.md`.

## Призначення роботи

Це домашнє завдання демонструє:

- використання **фільтра Блума** для перевірки унікальності без збереження всіх даних;
- застосування **HyperLogLog** для наближеного підрахунку кількості унікальних елементів у великих наборах;
- порівняння точних та ймовірнісних алгоритмів за:
  - часом виконання,
  - споживанням пам’яті,
  - допустимою похибкою.
